# Create a refund

Creates a refund for an already processed transaction. PayFlow notifies you
via `webhook_url` when the refund is processed, which may take several days
depending on the card network.

`POST /refunds`

## Body parameters

| Parameter            | Type   | Required | Description |
|----------------------|--------|----------|-------------|
| `transaction_id`     | string | Yes      | The ID of the transaction to refund. The transaction must be in a successfully processed state. |
| `amount`             | number | No       | The amount to refund in the smallest currency unit (e.g., cents). Must be less than or equal to the original transaction amount. If not provided, the full transaction amount is refunded. |
| `reason`             | string | No       | The reason for the refund, used for internal record-keeping. The customer does not see this value. Accepted values: `duplicate`, `fraudulent`, `customer_request`. |
| `refund_method`      | string | No       | Determines where the refunded amount is sent. `original` returns the funds to the original payment method. `wallet` credits the amount to the customer's PayFlow wallet — only available if the customer has an active PayFlow account. Defaults to `original`. |
| `webhook_url`        | string | No       | A URL that PayFlow calls with a POST request when the refund status changes. Must be a valid HTTPS URL. |

## Responses

| Status | Meaning |
|--------|---------|
| `201 Created` | Refund created successfully. Returns the refund object with the initial status `pending`. |
| `400 Bad Request` | The request body is missing a required field or contains an invalid value. Check the `error` field in the response for details. |
| `401 Unauthorized` | The API key is missing or invalid. |
| `404 Not Found` | The specified `transaction_id` does not exist. |
| `422 Unprocessable Entity` | The request is valid but cannot be processed — for example, if the refund amount exceeds the original transaction amount, or if the transaction is not in a refundable state. |

## Example request

```json
{
  "transaction_id": "txn_2c8f1a4e9b",
  "amount": 5000,
  "reason": "customer_request",
  "refund_method": "original",
  "webhook_url": "https://yoursite.com/webhooks/payflow/refunds"
}
```

## Example response — 201 Created

```json
{
  "id": "ref_7x2k9z1m",
  "transaction_id": "txn_2c8f1a4e9b",
  "amount": 5000,
  "status": "pending",
  "reason": "customer_request",
  "refund_method": "original",
  "webhook_url": "https://yoursite.com/webhooks/payflow/refunds",
  "created_at": "2026-06-25T14:30:00Z"
}
```
